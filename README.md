## 📋 Overview

**GreenIsland Online** is a comprehensive, full-stack travel and tourism management platform designed to streamline travel agency operations, trip management, and booking workflows. Built with **Angular 21.2.0** and powered by **ASP.NET Core Web API**, the platform provides an intuitive interface for managing complex travel logistics and customer interactions.

### 🎯 Key Features

#### 👥 Admin & Agency Management
- Role-based access control with comprehensive permission management
- Multi-level user hierarchy (administrators, agency staff, service agents)
- Secure authentication with encrypted credential handling
- Customizable menu permissions and access controls

#### ✈️ Travel Trip Management
- Advanced query system for travel requests (multi-step booking flow)
- Trip tracking and status management
- Comprehensive trip details including itineraries, destinations, and services
- Real-time search and filtering capabilities

#### 🏨 Inventory Management
- Hotel database with categories and pricing management
- Vehicle type tracking and allocation
- Room type configuration and availability
- Destination and location management
- Service inclusions/exclusions management

#### 💼 Business Operations
- Staff and employee management
- Department and designation structure
- Agency and company management
- Activity and service rate management
- Extra charges and special inclusions handling

## Development server

To start a local development server, run:

```bash
ng serve
```

Once the server is running, open your browser and navigate to `http://localhost:4200/`. The application will automatically reload whenever you modify any of the source files.

## Code scaffolding

Angular CLI includes powerful code scaffolding tools. To generate a new component, run:

```bash
ng generate component component-name
```

For a complete list of available schematics (such as `components`, `directives`, or `pipes`), run:

```bash
ng generate --help
```

## Building

To build the project run:

```bash
ng build
```

This will compile your project and store the build artifacts in the `dist/` directory. By default, the production build optimizes your application for performance and speed.
